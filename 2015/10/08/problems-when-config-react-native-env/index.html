react-native环境搭建问题

发表于 2015-10-08   |  

上个月Facebook开源了Android版的react-native，react-native为何物就不多介绍，个人认为虽然取代不了native，但是确实有可能是移动端的未来。

用这个新的工具最开始自然是需要搭建一个开发环境；官网说的可是简单：装好git, nvm等工具，两条命令解决：

npm install -g react-native-cli
react-native init AwesomeProject

但是对于生活在水深火热的天朝的程序员，事情远没有那么简单。如果你有一个稳定的VPN，那么可能就是几分钟的事情；但是openvpn已经斗不过GFW了，如果你用代理，那就绝对会掉进深坑。。。

node安装等问题

要安装node，最好安装一个node的版本管理工具；如果是macos还可以用brew直接安装；参照官网即可。安装node，需要翻墙，不然你的nvm install node会永远定住。我就忍不住吐槽了，我踏马nodejs怎么干什么了？GFW有什么理由墙？一群脑残2B。

假设我们装好了node, npm以及react-native那么，执行接下来一条命令很有可能遇到问题，初始化工程怎么都进行不了。因为初始化这个过程，需要用node的包管理工具下载一些依赖，但是很不幸，官方的源速度很慢甚至被墙；因此，如果不能科学上网，这个过程会无限延长甚至不能完成。如果选择代理。。

代理问题

代理是个大坑，等会说；先说说为什么VPN可以，代理有问题。

VPN和代理最大的区别是，VPN对于应用程序就相当于VPN躺在了TCP/IP协议栈里面，所有的网络请求都会通过VPN访问；而代理呢，我们需要给每个要用到代理的程序单独设置代理访问；大多数程序会检测诸如HTTP_PROXY的环境变量来自动使用代理，但是不是所有的程序都这么乖。

所以，如果你通过某种方式给系统设置了代理，并不意味着你的每一个程序都会通过代理访问；假如你设置了环境变量HTTP_PROXY那么，系统只是告诉程序有代理可用，至于用不用，是程序自己的问题；如果是VPN，那么程序就算什么都没干，它也是通过VPN访问网络的。

然后看看用代理会碰到什么问题，如果不解决这些问题，那么就算你有代理，就跟没有设置一样。

mac系统设置是全局代理吗

之所以提到mac，是因为react-native官方文档第一条：

OS X - Only OS X is currently supported

那么，我们打开系统偏好设置——网络——高级——代理 这个设置，然后配置好代理信息，对我们的安装有帮助吗？

事实上，终端以及一些基于命令行的工具，不会理会系统的代理设置；具体可以看看这里或者这里

所以，在系统这里设置代理对我们没有什么作用。

gyp的问题

你以为到这里就结束了？！ 实际上，我们使用的很多npm的包，用到了一些c/c++的模块，需要编译安装。这个时候，需要依赖node的源代码。但是，由于这个源本身的问题，有了代理速度还是乌龟一般。

没有办法，我们只有使用国内的淘宝镜像，先手动安装gyp。

openssl的问题

拜GFW所赐，使用https协议的话很有可能根本没法认证，所以最好吧官方的那个源换成http的；然后设置一下这个选项：
npm config set strict-ssl false

OK，这些问题全部解决的话，应该能顺利安装上react-native。

你当然可以选择使用淘宝的源安装这些依赖包，但是还是逃脱不了翻墙，换了个源只是让npm能顺利进行，安装过程中一些其他的工具呢，比如git，比如gyp源码安装等等。

代理这么麻烦，还有一些程序不听话，要是像vpn那样就好了。但是VPN就一定是最好的么？且不说现在VPN稳不稳定，由于是全局的，所以你访问国内的网站肯定没有原来快；如果你要频繁切换，麻烦不说，所有的程序都断线重连，也是蛋疼。
还是倾向与代理然后结合pac自动切换比较好，这样，我们只需要解决所谓的全局代理问题。解决方案自然是有，googleProxifier或者proxycap就行了。
